  • 安全注意事项

    • base64 (opens new window): base64 安全事项 (opens new window)，参见 RFC 4648 (opens new window)
    • cgi (opens new window): CGI 安全事项 (opens new window)
    • hashlib (opens new window): 所有构造器都接受一个 "usedforsecurity" 仅限关键字参数以停用已知的不安全和已封禁的算法 (opens new window)
    • http.server (opens new window) 不适合生产用途，只实现了基本的安全检查。 请参阅 安全性考量 (opens new window)。
    • logging (opens new window): 日志记录配置使用了 eval() (opens new window)
    • multiprocessing (opens new window): Connection.recv() 使用了 pickle (opens new window)
    • pickle (opens new window): 在 pickle 中限制全局变量 (opens new window)
    • random (opens new window) 不应当被用于安全目的，而应改用 secrets (opens new window)
    • shelve (opens new window): shelve 是基于 pickle 的因此不适用于处理不受信任的源 (opens new window)
    • ssl (opens new window): SSL/TLS 安全事项 (opens new window)
    • subprocess (opens new window): 子进程安全事项 (opens new window)
    • tempfile (opens new window): mktemp 由于存在竞争条件缺陷已被弃用 (opens new window)
    • xml (opens new window): XML 安全缺陷 (opens new window)
    • zipfile (opens new window): 恶意处理的 .zip 文件可能导致硬盘空间耗尽 (opens new window)

    -I 命令行选项可被用来在隔离模式下运行 Python。 当它无法使用时，可以使用 -P 选项或 PYTHONSAFEPATH 环境变量以避免在 sys.path 中预置一个潜在的不安全路径，如当前目录、脚本的目录或一个空字符串.
