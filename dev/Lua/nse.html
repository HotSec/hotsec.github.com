// 使用http-methods.nse脚本来枚举目标机器上所运行的服务
nmap -p 80,443 --script http-methods 192.168.31.1  

// 使用多个分类中的脚本对目标进行扫描
nmap --script discovery,intrusive 192.168.31.1

// 使用多个脚本对目标进行扫描
nmap --script filename1,filename2 192.168.31.1 

// 使用除了exploit、intrusive、dos分类以外的脚本对目标进行探测 
nmap --sV --script "not (exploit or dos or intrusive)" 192.168.31.1

// 支持通配符*
nmap --script "snmp-*" 192.168.31.1

// intrusive分类以外的 ftp-* 的脚本
nmap --script "ftp-* and not (intrusive)" 192.168.31.1

// 使用的时候传递参数
nmap -p 80 --script http-methods --script-args http.useragent="Mozilla 42" 192.168.31.1

// 从文件中载入脚本参数
nmap --script http-methods --script-args-file myargs.txt 192.168.31.1

# 1.3. nse实例：smtp-strangeport.nse

stmp-strangeport.nse 是一个简单的nse实例，实现的功能是 对目标主机的端口进行检测，查看是否有异常端口开放了SMTP服务（其实就是端口号不等于25、465、587），最后输出异常的端口。

description = [[
Checks if SMTP is running on a non-standard port.

This may indicate that crackers or script kiddies have set up a backdoor on the
system to send spam or control the machine.
]]

---
-- @output
-- 22/tcp  open   smtp
-- |_ smtp-strangeport: Mail server on unusual port: possible malware

author = "Diman Todorov"

license = "Same as Nmap--See https://nmap.org/book/man-legal.html"

categories = {"malware", "safe"}

portrule = function(host, port)
  return port.service == "smtp" and
    port.number ~= 25 and port.number ~= 465 and port.number ~= 587
    and port.protocol == "tcp"
    and port.state == "open"
end

action = function()
  return "Mail server on unusual port: possible malware"
end

# 1.4. NSE中的API

进行NSE开发的时，可以充分利用Nmap扫描网络时获得的关于主机和端口的信息。 NSE中的API就是连接脚本与扫描结果之间的桥梁。

# 1.4.1. Nmap API

Nmap中API的核心功能就是向脚本提供关于主机和端口的信息。

Nmap中的引擎会向脚本传递如下两个Lua table类型的参数： - host table - port table

可以用以下脚本来测试host 以及 port的值, line 17 return host 或者 retrun port

# 1.4.1.1. host table

  1. host.os
  2. host.ip
  3. host.name
  4. host.targetname
  5. host.directly_connected
  6. host.mac_addr
  7. host.mac_addr_src
  8. host.interface_mtu
  9. host.bin_ip
  10. host.bin_ip_src
  11. host.times
  12. host.traceroute 注意， 此字段只有在指定 --traceroute时才有用

# 1.4.1.2. port table

  1. port.number
  2. port.protocol
  3. port.service
  4. port.version
  5. port.state

# 1.4.3. NSE中的注册表

NSE中的注册表是一个lua table 类型的数据文件，它用来保存在一次扫描中各个脚本之间共享的变量， 这个注册表保存在一个名为Nmap.registry的变量中。

举个例子，在使用脚本对目标的口令进行暴力破解时，可以使用这个注册表报已经成功的用户名和密码保存起来，以供其他脚本使用。例如，破解到了目标的用户名为admin 密码为123456,NSE就会执行一个插入操作 table.insert(nmap.registry.credentials.http, {username = admin,passwdord = 123456 }
