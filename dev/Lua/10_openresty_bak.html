# 3. Nginx

Nginx是事件驱动型服务，注册各种事件处理器以处理事件。对于Nginx，事件的主要来源于网络和磁盘。事件evnet模块负责事件收集、管理和分发事件，其他模块都是事件的处理者和消费者，会根据注册事件得到事件的分发。

# 5. Nginx.conf

https://blog.csdn.net/jackliu16/article/details/79444327 (opens new window)

nginx配置文件：

  • # 单行注释
  • 单引号或者双引号 -- 定义字符串 \ 可以使用转义字符
  • 使用 $var 可以引用预定义的一些变量
  • 配置指令以分号结束，可以接受多个参数，用空白字符分隔
  • 配置块（block）是特殊的配置指令，它有一个 {} 参数而且无需分号结束， {} 里面可以书写多个配置指令，配置块也允许嵌套
  • 使用 include 指令可以包含其他配置文件，支持 * 通配符
  • 不能识别或错误的配置指令会导致nginx启动失败

# 6. 负载均衡

http {
    upstream backend {
        least_conn;        # 负载均衡策略
        server 127.0.0.1:80;  # 一台上游服务器
        server ... weight=3;    # 可以指定多台上游服务器
        server ... backup;      # 备份用的上游服务器    
        keepalive 32;       # 使用连接池，长连接复用
    }

    server {
        listen 8088;

        location /passto {
            proxy_set_header Host $host;   # 使用变量转发原始请求的host头部
            proxy_pass http://backend;  # 转发到upstream块定义的服务器集群
        }    
    }
}

# 14.1. 配置指令

  • lua_package_path 设置lua脚本的查找路径
  • lua_package_cpath 设置lua c模块（*.so)的查找路径
  • lua_code_cache 设置lua代码缓存，加速执行速度
  • lua_malloc_trim 定期清理内存，减少内存的占用
  • lua_shared_dict 定义一块共享内存
  • lua_check_client_about 检测客户端断连

lua_package_path/lua_package_cpath 以字符串的形式确定lua脚本和库的路径，多个路径用;分隔，默认的查找路径用;;。

# 14.2. 功能指令

xxx_by_lua

  • init_by_lua 在postconfiguration阶段执行lua代码

  • init_worker_by_lua 在init_process阶段执行lua代码

  • set_by_lua 在rewerite阶段操作Nginx变量

  • rewreite_by_lua 在rewrite阶段执行lua代码

  • access_by_lua 在access阶段执行lua代码

  • content_by_lua 在content阶段执行lua代码，产生响应内容

  • header_filter_by_lua 在header_filter阶段执行lua代码

  • body_filter_by_lua 在body_filter阶段执行lua代码

  • log_by_lua 在log阶段执行lua代码

  • balancer_by_lua 在负载均衡阶段执行lua代码

  • ngx_lua的功能指令通常都有三种形式

    • xxx_by_lua 执行指令后字符串形式的lua代码
    • xxx_by_lua_block 指令后是{...}的lua代码块
    • xxx_by_lua_file 执行磁盘上的脚本文件 （推荐）

# 14.4. 应用开发流程

  1. 确定对外服务端口（80？）,也就是定义Nginx的server
  2. 确定对外服务的URI,也就是定义的Nginx的location
  3. 确定执行业务逻辑的一个或多个具体阶段，，如rewrite/access/content等
  4. 在location里使用ngx_lua指令，配置基本参数和要执行的脚本
  5. 根据需求编写一个或多个lua脚本，实现业务
  6. 调试、测试，最后部署上线运行

# 15.4. 正则

  • ngx.re.matcch
  • ngx.re.find
  • ngx.re.gmatch
  • ngx.e.sub
  • ngx.re.gsub

使用这些函数应该尽量使用[[...]]的方式书写正则表达式，避免转义的问题;可以使用 jo 参数启用PCRE JIT编译。

-- 参数 "j" 启用 JIT 编译，参数 "o" 是开启缓存必须的
-- sub
local str = "abcdab-123ab"
str = ngx.re.sub(str,"ab","cd","jo")
ngx.say(str) -- cdcdab-123ab

-- gsub
local str = "abcdab-123ab"
str = ngx.re.gsub(str,"ab","cd","jo")
ngx.say(str) -- cdcdcd-123cd

-- find
local from, to = ngx.re.find(str,[[\d+]],"jo")
ngx.say(string.sub(str,from,to-1))

-- match
local regex = [[\d+]]
local m = ngx.re.match("hello, 1234", regex, "jo")
if m then
    ngx.say(m[0]) -- 1234
else
    ngx.say("not matched!")
end

-- gmatch
local it, err = ngx.re.gmatch("hello12331, wo424rld!as124", [[\d+]], "jo")
if not it then
    ngx.log(ngx.ERR, "error: ", err)
    return
end
while true do
    local m, err = it()
    if err then
        ngx.log(ngx.ERR, "error: ", err)
        return
    end
    if not m then
        -- no match found (any more)
        break
    end
    -- found a match
    ngx.say(m[0],"|",m[1]) -- 12331|nil
                           -- 424|nil
                           -- 124|nil
end

# 15.5. 请求处理

OpenResty/ngx_lua对http请求的处理提供非常完善的支持，大约有40个接口，可以操作请求行，请求头，请求体，响应头，响应体，执行重定向，跳转等。

# 15.5.1. 环境数据 ngx.ctx

表ngx.ctx是http请求的“环境数据”，相当与nginx中的r->ctx.它是每个请求所独有的，可以用来存储一些临时的数据，在整个处理流程中共享。

但是ngx.ctx使用成本较高，应当尽量少用。

# 15.5.2. 请求行

  • start_time 请求的开始时间，相当于$request_time
  • get_method 获取字符串形式的的请求参数
  • set_method 改写请求方法，使用数字常量，不能用字符串
  • set_uri 改写URI
  • get_uri_args 获取URI的参数，以表的形式返回解析后的结果
  • get_post_args 获取POST方式传递的参数，同样返回一个表
  • set_uri_args 改写URI参数

这些函数可以用rewrite阶段，改写URI的各种参数，实现跳转。

local arg = ngx.req.get_uri_args()
for k,v in pairs(arg) do
    ngx.say("[GET] key:", k, " v:", v)
end

ngx.req.read_body() -- 解析 body 参数之前一定要先读取 body
local arg = ngx.req.get_post_args()
for k,v in pairs(arg) do
    ngx.say("[POST] key:", k, " v:", v)
end

# 15.5.3. 请求头

  • raw_header 获取请求头的未解析字符串
  • get_headers 获取所有的HTTP请求头，返回一个表
  • set_header 改写一个HTTP请求头
  • clear_header 删除一个HTTP请求头

http的请求头都是Key-Value的形式，很适合用Lua里的表来存储，处理请求头的接口;后两个函数通常用在rewrite阶段，改写HTTP请求。

# 15.5.4. 请求体

  • discard_body 丢弃请求体
  • read_body 开始读取请求体的内容
  • get_body_data 获取请求体参数，是非阻塞的
  • set_body_data 改写请求体数据
  • get_body_file 如果请求体很大，存储在文件里，返回使用的文件名
  • set_body_file 把请求体数据写入文件，文件必须已经存在
  • init_body 创建一个新的请求体替代原请求的数据
  • append_body 向init_body创建的请求体里添加数据
  • finish_body 完成请求体数据的创建

# 15.5.5. 响应头

  • ngx.status 读取响应状态码
  • ngx.header 读写响应头
  • ngx.send_headers 发送响应头，是非阻塞的
  • ngx.headerrs_sent 检查响应头是否已经发送

ngx.header.content_length = len -- 相当于header['Content-Length']

# 15.5.6. 响应体

  • ngx.print
  • ngx.say
  • ngx.flush
location /test {
    content_by_lua_block {
        -- ngx.var.limit_rate = 1024*1024
        local file, err = io.open(ngx.config.prefix() .. "data.db","r")
        if not file then
            ngx.log(ngx.ERR, "open file error:", err)
            ngx.exit(ngx.HTTP_SERVICE_UNAVAILABLE)
        end

        local data
        while true do
            data = file:read(1024)
            if nil == data then
                break
            end
            ngx.print(data)
            ngx.flush(true)
        end
        file:close()
    }
}

# 15.6. 请求转发

corountine based socket功能以同步非阻塞的方式实现socket编程，高效地与任意后端服务通信。

内建连接池，实现长连接

支持tcp udp UNIX Domain Socket

函数 ngx.socket.tcp() 创建一个tcp协议的cosocket对象，能够与任意的TCP服务器通信，可以使用的方法有：

  • settimeout 设置tcp通信的超时时间
  • settimeouts 分别设置建立连接/读/写的超时时间
  • connect 建立有后端服务器连接
  • sslhandshake 发起SSL/TLS握手
  • send 发送数据
  • receive 接收数据
  • receiveuntil 以指定的模式持续接收数据
  • close 关闭连接
  • setkeepalive 把连接放入连接池，即长连接复用

ngx.socket.connect() 相当于ngx.socket.tcp()后紧接着调用connect(),返回一个已经成功创建连接的cosocket对象（tcp)

# 15.8. 定时器

定时器handler的执行是与HTTP请求处理分离的，所以在函数里不能使用ngx.var、ngx.say、ngx.print等请求相关的函数。

local delay = 5  -- 5s， 如果用毫秒可以用小数来指定
local handler
-- do some routine job in Lua just like a cron job
handler = function (premature)
    if premature then  -- 第一个参数必须是premature，函数可以由此判断Nginx worker进程是否正在退出，执行必要的清理动作
        return
    end
    local ok, err = ngx.timer.at(delay, handler)
    if not ok then
        ngx.log(ngx.ERR, "failed to create the timer: ", err)
        return
    end
end

local ok, err = ngx.timer.at(delay, handler)
if not ok then
    ngx.log(ngx.ERR, "failed to create the timer: ", err)
    return
end

# 15.9. 共享内存

ngx_lua的指令lua_shared_dict定义共享内存，格式如下(只能在http{}里使用)：

lua_shared_dict shm_name size # 定义一个大小为size名为shm_name的共享内存

lua_shared_dict stats 32m # 定义一个32m的共享内存

  • get 从共享内存里获取一个值
  • set 向共享内存里写入一个值，可以设置过期时间
  • add 类似set,但只有key不存在时才可以写入
  • replace 与add相反，不存在时才写入（替换？)
  • delete 删除共享内存里的一个key
  • incr 原子递增整数
  • lpush/rpush 队列操作，在两端添加数据
  • lpop/rpop 队列操作，在两端删除数据
  • llen 获得队列里的元素数量
  • flush_all 清空共享内存里的所有数据
  • flush_expired 清空共享内存里所有过期数据

ngx_lua的这个共享内存功能非常有用，极大的便利了worker进程间的通信和协作。

local stats = ngx.shared.stats  -- 获取之前定义的共享内存对象

stats:set("count",1234,10)     -- 在共享内存里设置一个值 count = 1， 10s后过期
local v = stats:get("count")  -- 获取共享内存里的值
local id = stats:incr("uid",0,1)  -- 原子增加一个整数
ngx.say("v: ",v)
ngx.say("id: ",id)
stats:rpush("sessions","xxx123xxx123xxx")  -- 队列操作，在右端插入一个元素
v = stats:lpop("sessions")              -- 队列操作，从左端取出一个元素
ngx.say(v)
stats:flush_all()           -- 清空共享内存里的所有数据
